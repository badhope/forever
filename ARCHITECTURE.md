# 项目架构

## 整体架构

Forever 采用前后端分离的架构设计：

```
┌─────────────────────────────────────────┐
│         Electron 桌面应用                │
│  ┌──────────────────────────────────┐  │
│  │      React + Ant Design          │  │
│  │      (渲染进程)                   │  │
│  └──────────────────────────────────┘  │
│                   ↕ IPC                 │
│  ┌──────────────────────────────────┐  │
│  │      Electron 主进程              │  │
│  └──────────────────────────────────┘  │
└─────────────────────────────────────────┘
                    ↕ HTTP/REST
┌─────────────────────────────────────────┐
│         FastAPI 后端服务                 │
│  ┌──────────────────────────────────┐  │
│  │      API 路由层                   │  │
│  │  - 文件管理 API                   │  │
│  │  - Git 操作 API                   │  │
│  │  - 插件管理 API                   │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│  │      核心业务层                   │  │
│  │  - LLM 统一调度                   │  │
│  │  - Git 仓库管理                   │  │
│  │  - 文件自动归档                   │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│  │      数据访问层                   │  │
│  │  - SQLAlchemy ORM                 │  │
│  │  - SQLite 数据库                  │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│  │      插件系统                     │  │
│  │  - Pluggy 钩子机制                │  │
│  └──────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

## 核心模块

### 1. LLM 统一调度 (llm.py)

支持 OpenAI 和 Ollama 两种后端，提供统一的接口：

- 文本生成
- 流式响应
- 模型切换
- 上下文管理

### 2. Git 仓库管理 (git_manager.py)

基于 pygit2 实现，支持：

- 仓库克隆
- 文件提交
- 分支管理
- 状态查询
- 远程同步

### 3. 文件自动归档 (file_organizer.py)

智能文件管理：

- 文件类型识别
- 自动分类
- 目录扫描
- 文件搜索
- 归档整理

### 4. 插件系统 (plugins/)

基于 pluggy 的可扩展架构：

- 钩子规范定义
- 插件加载/卸载
- 事件触发
- 生命周期管理

### 5. 数据模型 (models/)

SQLAlchemy 2.0 异步 ORM：

- File: 文件元数据
- Project: 项目配置
- Operation: 操作记录

## 数据流

```
用户操作
   ↓
Electron 渲染进程
   ↓
IPC 通信
   ↓
Electron 主进程
   ↓
HTTP 请求
   ↓
FastAPI API 路由
   ↓
核心业务逻辑
   ↓
数据库/文件系统/外部服务
```

## 扩展机制

### 添加新插件

1. 实现插件钩子接口
2. 在 `plugins/builtin/` 创建插件文件
3. 通过 API 或自动发现加载插件

### 添加新 API

1. 在 `app/api/` 创建路由文件
2. 在 `app/main.py` 注册路由
3. 定义 Pydantic 模型进行参数验证

### 添加新模型

1. 在 `app/models/` 创建模型文件
2. 在 `app/models/__init__.py` 导出
3. 数据库会自动创建对应的表
