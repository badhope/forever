MemGPT 式三层记忆架构

Core Memory — 核心记忆
始终在上下文中。角色身份、关键关系、重要偏好。容量有限但访问最快。
Recall Memory — 回忆记忆
可检索的近期对话记忆。通过 ChromaDB 向量搜索，语义匹配。
Archival Memory — 归档记忆
长期存储的大量记忆。反思洞察、深层认知。更低优先级但容量无限。

API 端点

Base URL: http://localhost:3000
方法路径说明
GET/health健康检查
GET/api/characters获取角色列表
POST/api/characters创建角色
GET/api/characters/:id获取角色详情
PUT/api/characters/:id更新角色
DEL/api/characters/:id删除角色
GET/api/sessions获取会话列表
POST/api/sessions创建会话
POST/api/sessions/:id/messages发送消息
DEL/api/sessions/:id删除会话

│   │   ├── security.ts       # 输入清理+Token计数
